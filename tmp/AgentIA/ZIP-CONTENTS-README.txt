📊 WHAT EACH AGENT DOES
═══════════════════════════════════════════════════════════════════════════

🟠 PRODUCT OWNER
   • Writes testable stories
   • Identifies blockers upfront
   • Defines clear acceptance criteria
   • Coordinates with Planner/QA

🔵 PLANNER
   • Validates stories (testability gate)
   • Breaks into tasks with test strategy
   • Coordinates QA + Dev + DevOps
   • Tracks blocker lifecycle

💚 DEVELOPER
   • Validates task clarity before coding
   • Codes following test strategy
   • Verifies Definition of Done
   • Reports blockers to Planner

🧪 QA AGENT
   • Creates detailed test plans
   • Identifies risks + edge cases
   • Tests systematically
   • Signs off on readiness

🟣 DEVOPS
   • Validates infrastructure readiness
   • Documents rollback plans (tested)
   • Validates systematically
   • Coordinates with developers

🏆 TECH LEAD REVIEWER
   • Reviews code independently
   • Validates architecture + security
   • Assesses performance impact
   • Final approval before merge


═══════════════════════════════════════════════════════════════════════════

🔄 7-PHASE WORKFLOW
═══════════════════════════════════════════════════════════════════════════

PHASE 1: PO defines story (with testable criteria)
PHASE 2: Planner validates + coordinates
PHASE 3: Dev + QA + DevOps work in PARALLEL
PHASE 4: QA tests systematically
PHASE 5: Tech Lead reviews code
PHASE 6: QA + Tech Lead approve? → MERGE
PHASE 7: Deploy + Monitor
