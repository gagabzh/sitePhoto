📊 PHASE-BY-PHASE WORKFLOW (6 Phases)
═══════════════════════════════════════════════════════════════════════════

PHASE 1: PRODUCT OWNER (Definition)
────────────────────────────────────

1. PO writes story:
   ✅ Titre clair, action concrète, user role, value
   
2. PO ajoute Acceptance Criteria:
   ✅ Testable (pas vague)
   ✅ Edge cases couverts
   ✅ Error states définis
   ✅ Performance scopée
   ✅ Accessibility + sécurité explicite
   
3. PO identifie Blockers (tech, product, QA)

4. PO attend QA sign-off ("testable, ready")

DELIVERABLE: Story avec acceptance criteria + blockers + QA sign-off


PHASE 2: PLANNER (Planning & Coordination)
───────────────────────────────────────────

1. Planner validates story (checklist):
   ✅ Criteria clear, QA approved, blockers identified
   
2. Planner breaks into tasks (with dependencies)

3. Planner coordinates avec QA:
   → "Quels risques? Quels edge cases durs? Quel setup?"
   
4. Planner définit Test Strategy per task

5. Planner coordonne avec DevOps (si infra needed):
   → "Infra ready? Dependencies? Rollback plan?"
   
6. Planner assigne tasks à Dev + DevOps:
   ✅ Task + Acceptance Criteria + Test Strategy + Gotchas
   ✅ Testability Score (1-25)
   ✅ Dependencies + Blockers

7. Planner tracks blocker lifecycle

DELIVERABLE: Task assignments avec test strategy + DevOps coordination


PHASE 3: DEVOPS (Infrastructure - PARALLEL with Dev)
──────────────────────────────────────────────────────

1. DevOps reçoit infra task (if needed):
   ✅ Valide readiness (10-item checklist)
   
2. DevOps documentes:
   ✅ Rollback plan (detailed, tested)
   ✅ Validation strategy
   ✅ Monitoring plan
   
3. DevOps implémente:
   ✅ Test on staging first
   ✅ Follow Definition of Done
   
4. DevOps valide infrastructure:
   ✅ Health checks pass
   ✅ Performance metrics normal
   ✅ Monitoring active
   
5. DevOps report to Planner:
   "Infrastructure ready for app deployment"

DELIVERABLE: Validated infrastructure, rollback tested


PHASE 4: DEVELOPER (Implementation - PARALLEL with DevOps)
───────────────────────────────────────────────────────────

1. Dev receives task:
   ✅ Valide clarity (10-item checklist)
   
2. Dev code:
   ✅ Follows patterns
   ✅ Writes tests per Test Strategy
   ✅ Edge cases + errors + security
   
3. Dev checks Definition of Done:
   ✅ All 10 items satisfied
   
4. Dev submit PR:
   ✅ Task code in title
   ✅ Test strategy summary
   ✅ Coverage percentage
   ✅ Blockers encountered

DELIVERABLE: PR ready for QA + Tech Lead


PHASE 5: QA VALIDATION (Testing)
─────────────────────────────────

1. QA receives PR:
   ✅ Follows Test Strategy
   
2. QA tests:
   ✅ Happy path + edge cases + errors + security
   ✅ Per test scenario from strategy
   
3. QA checks:
   ✅ All criteria met
   ✅ No regressions
   ✅ Infrastructure stable
   
4. QA report:
   ✅ "Ready to merge" OR
   🟡 "Blockers: [list]" with feedback

DELIVERABLE: QA approval (or feedback for fixes)


PHASE 6: TECH LEAD REVIEW (Code Review)
────────────────────────────────────────

Tech Lead reviews (independent of QA):
✅ Code quality (patterns, standards)
✅ Architecture (design decisions)
✅ Security (authentication, authorization)
✅ Performance (queries, caching)
✅ DevOps changes (if any: rollback tested?)

DELIVERABLE: Code approval (or feedback)


PHASE 7: MERGE & DEPLOYMENT
────────────────────────────

1. Both QA + Tech Lead approve? → Merge

2. Infrastructure is ready? → Yes (from Phase 3)

3. GitHub Actions auto-deploy:
   - Instance-1: app deployment
   - Instance-2: worker deployment (with auto-unshelve if needed)

4. Post-deployment:
   ✅ DevOps monitors
   ✅ QA spot-checks (no regressions)
   ✅ Planner unblocks dependent tasks

DELIVERABLE: Feature shipped, infrastructure stable
