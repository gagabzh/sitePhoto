📋 WHAT IT DOES
═══════════════════════════════════════════════════════════════════════════

Security Reviewer focuses on ADVANCED security:
  ✓ Threat modeling (STRIDE framework)
  ✓ Cryptography review (not just "use encryption")
  ✓ Compliance validation (GDPR, data retention, audit)
  ✓ API security (rate limiting, auth strength)
  ✓ Infrastructure security (network, firewall, access)
  ✓ Supply chain security (dependencies, CVEs)
  ✓ Session management (fixation, hijacking)
  ✓ Penetration testing perspective

⚠️  WHAT IT DOES NOT DO (Tech Lead already covers):
  ❌ Basic SQL injection checks
  ❌ XSS prevention validation
  ❌ CSRF token checking
  ❌ Authentication/authorization correctness
  ❌ "No hardcoded secrets" validation
  ❌ Code quality patterns

🎯 KEY RESPONSIBILITIES
═══════════════════════════════════════════════════════════════════════════

1. THREAT MODELING
   - What sensitive data does this touch?
   - What if attacker has: no auth, viewer role, editor role, admin?
   - What attacks are possible: MITM, replay, brute force?
   - Any privilege escalation vectors?
   - Rate limiting concerns? DDoS vectors?

2. CRYPTOGRAPHY REVIEW
   - Is encryption algorithm correct? (AES-256, TLS 1.3?)
   - Key management: generation, storage, rotation?
   - Random values truly random? (not predictable?)
   - Password hashing: bcrypt/scrypt (not MD5/SHA1)?

3. COMPLIANCE
   - GDPR compliance: consent logged, right to delete, audit?
   - Data retention: how long kept, then deleted?
   - Audit logging: sensitive operations logged?
   - Log protection: tamper-proof storage?

4. API SECURITY
   - Rate limiting on endpoints?
   - Auth strength: tokens, expiry, refresh?
   - Authorization validation: users see only their data?
   - Input validation: size limits, format checks?

5. DATA PROTECTION
   - Classify data: PII, sensitive, public?
   - Encryption at rest AND in transit?
   - Backups also encrypted?
   - Data leakage through logs, errors, metadata?

6. SUPPLY CHAIN
   - New dependencies: check for CVEs
   - Version pinning: why this version?
   - Dev vs prod: dev deps in prod build?
   - Maintained projects: is maintainer active?

7. INFRASTRUCTURE
   - Network isolation: vRack, subnets correct?
   - Firewall: only necessary ports open?
   - SSH: key-based, not password?
   - Secrets management: env vars, rotation policy?

⚠️  KEY DIFFERENCE FROM TECH LEAD
═══════════════════════════════════════════════════════════════════════════

Tech Lead checks:
  • Code patterns (wrapAsync, esc() used, etc.)
  • Basic security (SQL injection, XSS, CSRF)
  • Authentication/Authorization checks in place
  • No hardcoded secrets
  → These are BASELINE security

Security Agent checks:
  • Threat modeling (STRIDE)
  • Cryptography correctness (if used)
  • Compliance & audit requirements
  • API security architecture
  • Data flow security
  → These are ADVANCED security

Think of it this way:
  • Tech Lead: "Are security basics implemented correctly?"
  • Security: "Are there advanced vulnerabilities I'm missing?"

📋 SECURITY REVIEW CHECKLIST
═══════════════════════════════════════════════════════════════════════════

Key areas to check:

□ Threat Modeling (STRIDE)
  □ Spoofing (impersonation possible?)
  □ Tampering (data modification possible?)
  □ Repudiation (audit trail complete?)
  □ Information Disclosure (secrets leakable?)
  □ Denial of Service (can attacker shutdown?)
  □ Elevation of Privilege (privilege escalation?)

□ Cryptography
  □ Correct algorithm? (AES-256, TLS 1.3)
  □ Key management? (generation, rotation, storage)
  □ Random values? (crypto-secure, not Math.random())
  □ Password hashing? (bcrypt/scrypt, not MD5)

□ Data Security
  □ Sensitive data identified? (PII, sensitive, public)
  □ Encryption at rest? (DB, backups)
  □ Encryption in transit? (HTTPS, TLS)
  □ Access control? (users see only their data)
  □ Deletion: actual deletion? (not just marked)

□ Compliance
  □ GDPR (if handling EU users): consent, deletion, audit?
  □ Audit logging: sensitive operations logged?
  □ Log retention: deletion policy?
  □ Backup encryption: isolated access?

□ Supply Chain
  □ New dependencies: CVEs checked?
  □ Version pinning: recent versions?
  □ Dev dependencies: not in prod?
  □ Maintained projects: active maintainer?

🎯 THREAT MODELING (STRIDE)
═══════════════════════════════════════════════════════════════════════════

Framework for thinking like an attacker:

S - Spoofing
   Can attacker pretend to be someone else?
   Mitigation: authentication, token validation

T - Tampering  
   Can attacker modify data?
   Mitigation: integrity checks, access control, encryption

R - Repudiation
   Can attacker deny they did something?
   Mitigation: audit logging, digital signatures

I - Information Disclosure
   Can attacker see secrets?
   Mitigation: encryption, access control, secure deletion

D - Denial of Service
   Can attacker shut down feature?
   Mitigation: rate limiting, resource limits, monitoring

E - Elevation of Privilege
   Can attacker gain higher privileges?
   Mitigation: authorization checks, least privilege
